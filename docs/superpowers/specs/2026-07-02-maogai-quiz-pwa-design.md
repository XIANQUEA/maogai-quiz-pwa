# 毛概期末刷题 PWA 设计说明

日期：2026-07-02  
状态：已完成对话确认，待用户审阅书面规格

## 1. 项目目标

制作一款面向手机、单人本地使用的毛概期末刷题应用。用户通过浏览器访问后可将其添加到手机桌面；首次加载完成后，断网仍可练习。应用无需登录、服务器或云数据库。

题库来源为 `24毛概练习题整合版.doc`。原文约 4.8 万字符，包含导论、第一至第八章和结束语，共 10 个部分；每部分包含判断题、单项选择题、多项选择题和参考答案。原文存在少量编号、空格及答案标记不统一的问题，正式导入前必须清洗并列出异常项。

## 2. 已确认的产品范围

### 2.1 首版包含

- 章节练习
- 随机练习，支持抽取 10、20 或 50 题
- 错题本
- 判断题、单选题、多选题
- 提交后立即判题并显示标准答案
- 已练题数、正确率、错题数和章节进度统计
- 本地记录清空
- 学习记录 JSON 备份导出与导入恢复
- PWA 安装及离线使用
- “清爽学习蓝”手机界面

### 2.2 首版不包含

- 登录、云同步、多人数据和排行榜
- 模拟考试、倒计时和统一交卷
- 题目解析
- 在线题库编辑后台
- 后端服务器

## 3. 核心页面与用户流程

### 3.1 首页

首页显示总题数、已练题数、总体正确率、当前错题数和本周练习进度，并提供三个主要入口：

1. 章节练习
2. 随机练习
3. 复习错题

底部导航固定为：首页、练习、错题、设置。

### 3.2 章节练习

用户先选择章节，再选择判断题、单选题、多选题或全部题型。系统按原题库顺序生成本次练习。已完成的题目仍可重新练习，历史统计累计保留。

### 3.3 随机练习

用户选择 10、20 或 50 题。系统从全部题库中无重复抽取；题量大于可用题数时，自动使用全部可用题目并给出提示。

### 3.4 答题页

- 每次只显示一道题，降低手机屏幕的信息负担。
- 顶部显示章节、题号、总题数、进度条和收藏按钮。
- 判断题显示“正确/错误”两个大按钮。
- 单选题使用单选交互。
- 多选题使用复选交互，并明确提示“多选题需全部选对”。
- 底部主按钮固定为“提交答案”；判题后变为“下一题”。
- 未选择任何答案时不可提交，并显示轻量提示。
- 提交后锁定选项，显示正确/错误状态和标准答案。

### 3.5 练习结果

一组练习结束后显示题量、正确数、错误数、正确率和本组错题列表，并提供“重做本组错题”和“返回首页”。

### 3.6 错题本

错题可按章节和题型筛选。普通练习答错后，题目自动进入错题本并累计错误次数；在错题本中答对一次后标记为“已掌握”，从当前错题列表移除，但历史错误次数继续保留。以后再次答错时重新进入错题本。

### 3.7 设置

- 导出学习记录
- 导入学习记录
- 清空全部学习记录
- 查看题库版本和应用版本
- 查看安装到桌面及离线使用说明

清空记录必须经过二次确认。

## 4. 视觉与交互设计

采用已确认的“清爽学习蓝”方向：

- 页面背景：接近白色的浅蓝灰
- 主色：清晰的中亮蓝
- 成功状态：绿色
- 错误状态：红色
- 内容以白色圆角卡片承载
- 字体优先使用系统无衬线字体，保证不同手机的加载速度和可读性
- 主要点击区域不小于 44×44 CSS 像素
- 底部操作区适配 iPhone 安全区域
- 正确与错误状态同时使用颜色、图标和文字表达，不能仅依靠颜色
- 最小适配宽度为 360 CSS 像素

首页强调“继续练习”，答题页强调单题专注和单手操作，不使用复杂动画。仅保留短促的选项反馈和页面切换过渡。

## 5. 技术架构

首版采用原生 HTML、CSS 和 JavaScript 构建静态 PWA。

### 5.1 模块边界

| 模块 | 职责 | 主要依赖 |
| --- | --- | --- |
| App Shell | 页面框架、路由、底部导航和全局状态协调 | 各业务模块 |
| Question Repository | 加载、校验、筛选题库 | 版本化题库 JSON |
| Session Engine | 创建章节、随机和错题练习会话 | Question Repository |
| Grading Engine | 判断题、单选题和多选题的确定性判分 | 无外部依赖 |
| Progress Store | 保存答题历史、错题状态和统计 | IndexedDB |
| Backup Service | 导出、校验、导入本地记录 | Progress Store |
| PWA Service | 安装清单、缓存与离线启动 | Service Worker |
| UI Components | 题目卡片、选项、统计卡、反馈和对话框 | App Shell |

各模块通过明确的数据对象通信。判分逻辑不读取页面元素，进度存储不直接操作界面，从而便于独立测试和后续替换实现。

### 5.2 题库数据模型

每题至少包含：

```json
{
  "id": "chapter-00-single-004",
  "chapterId": "chapter-00",
  "chapterTitle": "导论",
  "type": "single",
  "sourceNumber": 4,
  "stem": "题干",
  "options": [
    { "key": "A", "text": "选项内容" }
  ],
  "answer": ["C"],
  "bankVersion": 1
}
```

`type` 只允许 `judgment`、`single`、`multiple`。答案统一存为数组：判断题使用 `["T"]` 或 `["F"]`，单选为一个字母，多选为排序后的多个字母。

题目 ID 由章节、题型和原题号组成，在同一来源题库内保持稳定。题库更新时不得因显示顺序变化而重建已有 ID。

### 5.3 学习记录模型

每题记录：

- 累计作答次数
- 累计正确次数
- 累计错误次数
- 最后一次作答时间
- 最后一次答案
- 当前是否属于错题
- 是否已掌握

另存练习会话摘要，用于首页和结果页统计。统计数据以题目记录为事实来源，可在升级后重新计算，避免缓存统计与真实记录不一致。

## 6. 数据流

1. 应用启动后载入 App Shell。
2. Question Repository 校验题库版本和数据结构。
3. Progress Store 打开 IndexedDB 并读取本机进度。
4. 首页根据题库和进度计算统计。
5. 用户选择练习模式后，Session Engine 生成不重复的题目序列。
6. 用户提交答案后，Grading Engine 返回判定结果。
7. Progress Store 以一次事务更新题目记录和会话摘要。
8. 页面显示判题反馈；用户进入下一题。

## 7. 离线与更新策略

- 使用 Web App Manifest 提供名称、图标、主题色和独立窗口启动方式。
- Service Worker 缓存 App Shell、样式、脚本、图标和版本化题库。
- 首次完整加载成功后才显示“已可离线使用”。
- 静态资源采用版本化缓存；新版本安装完成后提示用户刷新。
- 应用更新只替换程序和题库缓存，不清除 IndexedDB。
- 默认部署到 GitHub Pages；构建产物同时保持为通用静态文件，可迁移到其他 HTTPS 静态托管。

## 8. 错误与异常处理

### 8.1 题库导入

导入脚本必须输出：

- 总章节数及各题型题数
- 缺失选项
- 缺失答案
- 答案包含不存在选项
- 重复题目 ID
- 无法识别的题号或题型

存在上述结构性错误时不得生成正式题库。格式空格可自动清洗；答案内容以源文档为准，应用不擅自改写知识性答案。

### 8.2 本地存储

IndexedDB 打开失败时，应用进入“临时练习模式”，允许答题但明确提示记录不会保存。写入失败时保留当前页面状态并允许重试。

### 8.3 备份恢复

备份文件包含格式版本、题库版本、导出时间和学习记录。导入前完成结构校验；导入采用单一事务，任何校验或写入失败都不得覆盖原记录。成功导入后重新计算统计。

## 9. 测试策略

### 9.1 单元测试

- 判断题判分
- 单选题判分
- 多选题顺序无关但必须完全匹配
- 随机抽题无重复
- 题量不足时的降级行为
- 错题进入、掌握、再次进入
- 正确率和章节进度计算
- 备份格式校验和合并事务

### 9.2 集成测试

- 从选择章节到完成一组练习
- 随机练习完整流程
- 错题复习完整流程
- 刷新、关闭并重新打开后记录仍存在
- 导出后清空记录，再导入恢复
- 应用升级后保留原记录

### 9.3 手机与离线验收

- 360px、390px 和 430px 宽度下无横向滚动或内容遮挡
- 安卓 Chrome 可安装并独立窗口启动
- iPhone Safari 可“添加到主屏幕”
- 微信内置浏览器显示“在系统浏览器中打开”的安装提示
- 首次在线加载完成后，开启飞行模式仍可进入应用、载入题库并完成答题

## 10. 完成标准

首版只有在以下条件全部满足后才算完成：

1. 10 个部分的题目、选项和答案均已结构化导入。
2. 所有题库结构异常均已解决或由用户确认。
3. 三种题型判分结果与题库答案一致。
4. 章节练习、随机练习和错题本均可完整使用。
5. 学习记录在刷新和重启后保留。
6. 备份能够导出、清空后重新导入并恢复一致数据。
7. 断网可启动和练习。
8. 目标手机宽度下页面可读、按钮可触达且无布局缺陷。
9. 不需要账号或后端即可部署和使用。
